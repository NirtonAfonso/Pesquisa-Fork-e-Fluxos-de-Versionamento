O fork é usado para criar uma cópia de um repositório na sua conta, ideal para colaborar em projetos que você não pode editar diretamente. Depois do fork, o próximo passo é fazer o clone, que traz o projeto para sua máquina, diferente do branch, que é apenas uma linha paralela de desenvolvimento dentro do próprio repositório. Sobre fluxos de trabalho, o GitHub Flow é simples e ágil para entregas contínuas, o Git Flow é mais estruturado e adequado a equipes maiores com releases definidas, enquanto o Trunk-Based Development foca em integrações rápidas com a branch principal. O fluxo básico de colaboração envolve: fork, clone (por exemplo, usando GitHub Desktop), commits pequenos e frequentes, push das alterações e, por fim, abertura de um Pull Request para revisão e integração.
