O fork serve para criar uma cópia completa de um repositório em sua conta, ideal para contribuir em projetos nos quais você não tem permissão direta de edição. Após o fork, o passo seguinte é realizar o clone, que baixa o projeto para sua máquina - diferente do branch, que representa uma linha paralela de desenvolvimento dentro do mesmo repositório. Em relação aos fluxos de trabalho, o GitHub Flow é simples e voltado a entregas contínuas, o Git Flow é mais detalhado e indicado a equipes maiores com versões definidas, enquanto o Trunk-Based Development prioriza integrações rápidas na branch principal. O processo colaborativo básico inclui: fork, clone (por exemplo, via GitHub Desktop), commits pequenos e constantes, push das alterações e, por fim, abertura de um Pull Request para revisão e integração.
