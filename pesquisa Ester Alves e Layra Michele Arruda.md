# Pesquisa — Fork e Fluxos de Versionamento

**Autores:** Ester Alves de Jesus, Layra Michele Arruda Ferreira Cardoso
**Turma:** 2º ano do Ensino Médio

**Texto (5–8 linhas):**  
Fork é a criação de uma cópia de um projeto (código-fonte) existente para desenvolver uma nova versão independente, podendo ser usados quando uma pessoa quer contribuir para um projeto aberto.Ex: você quer adicionar uma nova função numa calculadora simples,
então faz um fork e cria um novo projeto com a mesma base. Sendo assim, o fork cria uma cópia completa de um repositório para outra conta, o clone cria cópia local de um repositório para o computador e o branch, é uma ramificação dentro do mesmo repositório.
O GitHub Flow tem lançamentos formais e estruturados (projetos com "releases" formais e agendadas), o Git Flow possui agilidade e Deploy Contínuo (Deploy frequente) e o Trunk-Based Development tem integração e velocidade máxima (Máxima velocidade de entrega).
Um Pull Request é um pedido formal para integrar alterações de uma branch em outra, sendo crucial para revisão de código e garantia de qualidade em projetos colaborativos.
