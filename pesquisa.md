# Pesquisa — Fork e Fluxos de Versionamento

**Autores:** Nataly Geovanna  da Matta Santos Evangelista, Eloise da Silva Santos
**Turma:** 2º ano L do Ensino Médio

**Texto (5–8 linhas):**  
Um fork é uma cópia de um repositório usada para modificar sem afetar o original, como quando queremos contribuir com um projeto open source. O fork cria uma cópia online, o clone baixa o projeto para o computador e o branch cria uma linha de trabalho separada. O GitHub Flow é simples e ideal para deploys frequentes, o Git Flow organiza versões e lançamentos formais, e o Trunk-Based Development integra mudanças rápidas em uma única branch principal. O Pull Request permite revisar e aprovar alterações antes de juntá-las ao código principal, garantindo colaboração e qualidade no projeto.
