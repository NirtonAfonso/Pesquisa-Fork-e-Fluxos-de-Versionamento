# Pesquisa — Fork e Fluxos de Versionamento

**Texto (5–8 linhas):**  
Um fork é uma cópia completa de um repositório que permite a qualquer usuário modificar, testar ou adicionar funcionalidades sem afetar o projeto original. Ele é especialmente útil em projetos colaborativos e open source, onde várias pessoas podem trabalhar em melhorias 
de forma independente. Por exemplo, um desenvolvedor pode fazer um fork de um repositório de um site público, implementar novas funcionalidades ou corrigir bugs e, depois, enviar essas alterações para o repositório original através de um Pull Request. A diferença entre 
fork, clone e branch está na forma como se trabalha com o projeto: o fork cria uma cópia independente do repositório online, o clone baixa todos os arquivos e histórico para o computador local, e o branch cria uma ramificação dentro do próprio repositório para desenvolver 
uma funcionalidade ou correção sem impactar o código principal. Em relação aos fluxos de trabalho, o GitHub Flow é simples, baseado em uma branch principal e ramificações curtas, ideal para equipes pequenas e deploys frequentes; o Git Flow é mais estruturado, 
com branches de desenvolvimento, release e hotfix, adequado para projetos que exigem versões formais e controle de releases; já o Trunk-Based Development prioriza a integração constante de pequenas alterações em uma única branch principal, sendo ideal para equipes 
maduras que possuem automação e integração contínua. O Pull Request é uma ferramenta essencial na colaboração, pois permite que outros membros da equipe revisem, comentem e aprovem mudanças antes de serem incorporadas ao código principal, garantindo qualidade, 
segurança e uma comunicação eficiente entre todos os desenvolvedores envolvidos no projeto.
