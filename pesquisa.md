O fork cria uma cópia de um repositório na sua conta GitHub. É útil para testar, modificar ou melhorar o código sem afetar o original. Use quando quiser contribuir com projetos open source ou não tiver acesso direto ao repositório. Também é ótimo para experimentar ideias sem bagunçar o projeto principal. Exemplo: você faz um fork de uma lista de tarefas, adiciona lembrete por e-mail e sugere via Pull Request. Fork ≠ clone ≠ branch: fork copia o projeto online, clone baixa localmente, branch cria uma linha paralela. GitHub Flow é ágil e direto; Git Flow é ideal para projetos com versões formais; Trunk-Based foca em integração contínua. Pull Requests são essenciais para revisar código, colaborar com segurança e manter a qualidade do projeto.
