Fork é uma cópia independente de um repositório que permite que você faça mudanças sem afetar o projeto original, sendo útil quando se quer contribuir para projetos de outras pessoas ou experimentar sem risco. Por exemplo, ao encontrar um projeto público no GitHub, você pode fazer um fork para implementar uma nova funcionalidade. Clone é baixar uma cópia local do repositório para seu computador, enquanto branch é uma ramificação dentro do mesmo repositório para desenvolver funcionalidades separadamente. GitHub Flow é um fluxo simples, ideal para deploy frequente, onde cada mudança é feita em uma branch e integrada rapidamente via pull request. Git Flow é mais estruturado, com branches para releases e hotfixes, adequado para projetos com ciclos de release formais. Trunk-Based Development envolve poucos branches, incentivando integrações frequentes, sendo bom para equipes pequenas e deploy contínuo. Pull Request é uma solicitação para revisar e integrar mudanças em um branch principal, garantindo qualidade, revisão e colaboração antes da fusão do código.
