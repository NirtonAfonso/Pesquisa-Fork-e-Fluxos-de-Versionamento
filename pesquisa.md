# Pesquisa — Fork e Fluxos de Versionamento

**Autores:** João Miguel, Joao nery  
**Turma:** 2º ano do Ensino Médio


Fork é uma cópia independente de um repositório no GitHub que permite modificar o projeto sem afetar o original; por exemplo, você faz um fork do repositório do professor e trabalha nele.  
Clone é a cópia desse repositório para o seu computador; branch é uma linha separada de desenvolvimento dentro de um repositório.  
GitHub Flow usa branches curtas a partir da main e merge rápido — bom para deploys frequentes; Git Flow organiza feature/release/hotfix e é indicado para projetos com releases formais; Trunk-Based privilegia commits pequenos e integração contínua, útil para times pequenos.  
Pull Request é a solicitação para integrar suas mudanças ao repositório alvo; serve para revisão de código, discussão e controle de qualidade antes do merge.
